Descovrir Style Guide & Component Library

Welcome to the Descovrir Style Guide. This page covers the styling and conventions of Descovrir user interfaces. This guide also include commonly used components. Suggestions are welcome via pull requests.

1. Layout, Grid, and Rhythm

The Descovrir layout system is derived from the Frameless Grid system. Some naming conventions and niceties are borrowed from the Twitter Bootstrap.

Descovrir uses a larger than typical default metric system. The base font size is 16px, with a line-height of 24px. The basic grid rhythm is established at 12px. Columns default to 48px, and gutters default to 24px. Because of Descovrir's large metrics, a great deal of focus and minimalism will be required.

Different screen sizes will alter the number of columns present. The smallest size is 4 columns with 5 gutters, which is 312px. The next size is then 6 (456px), then 8 (600px), then 12 (888px), and the largest size is at 16 columns (1176px). Rows are simply given the class row. Columns are classed as col-n, where n is the number of columns.

Columns in groups of two are preferred to one or odd numbers of columns.

Buckets can be targeted towards width of 4, 6, 8, 12, and 16 specifically. It uses the name col-tm-n, where t is literal, m is the target width, and n is the number of columns. The target will apply to all smaller sizes until specified. Columns can be nested by nesting rows. Columns may be offset using the name col-offset-n, where n is the number of columns to offset.

Descovrir uses a color scheme derived from HUSLp, which corrects issues with HSL, CIE-based color schemes, and HUSL itself at the price of highly saturated colors.

Hue. 5 primary hues are used by Descovrir.

  • 9° – Red. Reserved for errors, danger, major issues, or to provide warmth.
  • 81° – Gold. Serves as the base hue. Useful for warning states. Where possible use this hue over red.
  • 153° – Green. Useful for success and acceptance states.
  • 225° – Blue. Indicates a clickable element or navigation element. Also an accent color when no other is applicable.
  • 297° – Purple. Represents authority and acceptance.

Other hues can be used, particularly when hue is a linear function, such as a progress bar color.

Saturation. Descovrir defaults to using the full saturation available (100%) at any particular hue. Desaturation is available particularly at disabled states, and multiple of 20% are preferred for such states.

Lightness. Any lightness values may be used. Descovrir recommends against values darker than 20%. Default to 20%, 43%, 66%, 90%, 95%, and 99% where applicable.

2.2 Icons

2.2.1 Icon font

Descovrir will use FontAwesome where applicable. Use the i tag for icons, not italics.

2.2.2 Region flags

Where applicable, we will use this collection of flags for countries and regions.

2.3 Images

Alt tags, title tags, and captions for images are recommended where possible.

2.3.1 Image sizes

Images subscribe to the basic grid of 12px as much as possible, more so on the width. Additionally, widths that match the grid system, 48n + 24(n-1), are preferable.

2.3.2 Image captions

Images can be framed by wrapping them in a figure tag. Be sure to subtract 24px to account for padding incurred.

The figcaption tag can be used inside of a figure tag to add a caption. This prodecure is useful but not limited to images.

2.3.2 Avatars

Avatars will simply use the Gravatar system. Sizes match the image sizes stated above. Avatars are not be captioned.

2.3.2 Logos and Branding

The Descovrir logo is a mariner's astrolabe. It is presented on all pages, and clicking it either returns to a home page or presents an overlayed menu of options.

The Descovrir name, when used independently, is lowercase, normal weight Palatino in color #343115. It is not required to accompany the logo. When used in a sentence or title, Descovrir is capitalized.

3. Typography

3.1 Font stacks

Descovrir's primary typeface is Palatino. Monospaced typography uses DejaVu Sans Mono derivatives. When necessary, Lucida can be used as a san-serif typeface. Note Lucida does not support true italics on several operating systems.

  • Base: "palatino linotype", palatino, palladio, "urw palladio l", "book antiqua", serif
  • Monospace: menlo, "dejavu sans mono", consolas, "lucida console", monaco, monospace
  • Sans-serif: "lucida grande", "lucida sans unicode", "lucida sans", geneva, sans-serif

The classes base-font, monospace, and sans-serif can be used to alter the typeface.

3.2 Headings, Subheadings, and Header Groups

All elements h1 through h6 are styled.

Sub-headers use the class subheader when used directly below a higher order header. An hgroup will collapse the margins between the headers.

3.2.1 Small inline elements with headers

Optionally, elements can float along with the header. The most easy way to accomplish this is to wrap the header in an hgroup and add the class inline-block to the header tag.

4. Inline-level tags

4.1 Links

Links are simply blue in color, and like other states, lighten on hover and focus, and darken on active (clicked) or selected. Icons can be optionally included in links. Links stay underlined to support the color. Visited links receive no extraordinary treatment.

  • Hovered or focused link

4.2 Small, Strong, Emphasis, Deletion, Insertion, Mark, Subscript, Superscript, Abbreviations, and Code

All normal tag operations are supported and provide expected results. Note that b, s, and u tags are not supported, and i tags are reserved for icons, not italics.

  • An emphasized element

Other elements, such as var, samp, and kbd, are styled but non-effectively.

4.3 Defintion

The dfn element can be used on the defining instance of a term. It is often used with a title attribute with a defintion of the term.

4.3 Time and Dates

The time tag is used to describe specific dates and times. It does not indicate durations. The datetime attribute is optional.

4.4 Inline Quotations and Citations

The q and cite tags are used together to indicate a quote and a citation. All quotes are cited. No quote marks are included when using the q tag.

4.5 Inline Labels and Badges

Labels and badges are available in each of the basic hues.

  • Base or Warn: label and badge: 100

5. Block-level tags

5.1 Paragraphs

Paragraphs follow standard conventions. Paragraphs have a default maximum width to improve readability. This can be overwritten using the full-width class.

5.1.1 Leading Paragraphs

Leading paragraphs, commonly used after the page title, emphasis a block of text. Often leading paragraphs are used to describe the page contents.

5.3 Quotes and Citations

In addition to inline quotations, blockquotes can be used to pull a quote out of inline copy. The q and cite tags are used inside the blockquote tag.

5.4 Horizontal Rules

Horizontal rules are avoided as Descovrir prefers whitespace over lines. Rules are useful in certain situations, however.

5.5 Addresses, Header, Footer, Aside, Section, Article, Nav

Addresses are commonly misunderstood elements. The address tag is used to indicate who to contact for a particular document. The tag is not used for arbitrary postal addresses and email address. Descovrir does not style the address tag.

Other block tags such as header and footer are used for semantic representation, but receive no special styling.

5.6 Programming Code and Preformmated Elements

Preformmated element will default to use monospaced fonts. Computer code is syntax highlighted based on language.

7. Forms

7.1 Basic Forms

Forms are perhaps the most complicated part of any web design. The Descovrir style guide provides a usable, consistent, comprehensive approach to form elements.

A basic form

A description of a basic form.

Your name belongs in the box

Most standard forms start with a title and short description of the form.

Form fields are wrapped in a class of form-field and the type of the form field, such as form-field--text. Forms will typically have labels above the field and descriptions below the field, validation messages to the right of the field, and only a single column of fields. Most forms terminate with one, and only one, primary action button and several possible secondary actions. Form fields may have the following elements:

  • Hover, focus, active, success, information, warning, error, and disabled states and messaging
  • Icons prepended or appended to the field
  • Field labels
  • Required or optional labels
  • Field descriptions
  • Placeholder attribute

7.3 Form Field Labels and Descriptions

7.3.1 Field labels

Every field that is not in a horizontal form has a label. The label should click to focus on the related form element.

7.3.2 Optional and Required Fields

The label may contain a span with either optional or required as the class and copy to indicate the field is either required or optional. More useful for forms with more than three fields or several sections.

7.3.3 Field Descriptions

In the standard vertical layout, each form field may have a short description below the field. These descriptions are particularly useful when the user may not understand the expected input, or has reservations about why the field is necessary or benefitial.

We need a name to identify you to other users.

7.3.4 Fields with Icons

Icons can be prepended (default) or appended to fields. Icons can be useful particularly in situations where the placeholder will be blocked.

7.4 Fieldsets and Legends

Many users have difficult with forms with more than three fields. In such situations, it is useful to group the fields into fieldsets and legends to segment the field. These fieldsets can also be constructed as a step-by-step wizard.

11. Extentions and Additions

11.1 Extending with BEM

Non-tag elements in the Descovrir ecosystem use the BEM syntax.

11.2 JavaScript and CSS classes

Classes and IDs used only for JavaScript are prepended by js-. Style-based classes, other than for forms, navigation, and elements should not be used in JavaScript.

11.3 Additions

Prefer to add components to the general styleguide than inventing new components for specific interfaces. Reuse will ensure code maintainability and efficiency.

11.4 Browser support

At this time, Descovrir will support the major browsers at their current version, and one previous version.
